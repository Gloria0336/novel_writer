# Tower — 混合制策略遊戲｜設計總覽

## 願景

一款**混合制**策略遊戲：**回合制的戰略層**（每月一輪、四階段同時行動）+ **即時的戰術層**
（接戰時打小規模即時戰鬥），兩層共用**一張連續六角地塊地圖**。玩家可在回合內**發展科技 / 進化
流派**、**建設城池、鋪路造橋、改造地形**，再於戰術層把這些建設與地形變成實際戰場優勢。
參考典範：《全面戰爭》的雙層結構。

異世界劍與魔法：只有女性、天賦強大的**人類**（玩家），對抗數量龐大、棲於複雜巢穴的
**魔物**（AI）。人類少而堅、魔物多而脆，是一場質與量的非對稱戰爭。

> 本遊戲為獨立專案，與 `novel_db` / opera 無關，不讀寫小說資料。
> 本設計由「回合制網狀佔點」改造為混合制，改造路線圖與決策記錄見 [09](09-hybrid-redesign-plan.md)。

## 核心迴圈：戰略回合（每月四階段）+ 即時戰術戰鬥

每一輪 = 一個月，依序四階段，**雙方同時提交、系統收齊後結算**（細節見 [06](06-turn-and-phases.md)）：

1. **內政** `DOMESTIC` — 資源分配 / 科技研發 / 產生兵力 / **建築 / 鋪路造橋 / 改造地形**（進建造佇列）。
2. **情報** `INTELLIGENCE` — 各自探查對方部分行動；視野以地塊半徑揭露，情報可能模糊或錯誤（[08](08-intelligence-and-fog.md)）。
3. **軍事** `MILITARY` — 軍團在連續地圖上移動（移動點 + 地塊成本）；提交部署意圖。
4. **戰鬥結算** `COMBAT` — 找出接戰點，逐一切出戰場進入**即時戰術戰鬥**（[10](10-tactical-combat.md)）；
   玩家方可陣型半自動微操，其餘 AI 自動。收齊結果回填傷亡 / 佔領。

→ 月末完成到期建造、檢查全域勝負，否則進入下一個月。

## 勝負

- 人類勝：佔領魔物**主巢**或殲滅全部巢穴。人類敗：**王城**被佔。魔物方反之。

## 技術棧（推薦）

- **戰略引擎：Python 3.14 + Pydantic v2** — 回合制數值 / 規則模擬與 AI 盤面評估；Pydantic 模型
  同時是資料 schema 與引擎型別基礎。
- **戰術引擎：固定步長即時模擬** — `dt` 固定邏輯幀 + seeded RNG，達成可重現 / 可重播；
  玩家下 unit-group 指令（陣型半自動），或 AI 在同引擎自動下令（`AUTO_RESOLVE`）。
- **確定性**：單一 `GameState.master_seed`，以 `derive_seed()` 衍生地圖生成 / 每場戰術戰鬥 /
  每次情報 / 每個突發事件的獨立子 seed，全局可重現。
- **設定資料：YAML**（`../data/*.yaml`）— 魔物 / 科技 / 單位 / 加成 / 建造範本皆資料化。
- **介面：React + Vite + TypeScript** — hex 地圖渲染、戰術戰場渲染、科技樹與建造 UI，消費引擎 JSON。

## 一個月走查範例（混合制文字版）

驗證戰略四階段、即時戰術戰鬥、迷霧情報、建造改造彼此自洽。
假設 `master_seed="demo"`，已生成一張連續地塊地圖，時值 `month=3`。

**起始局面**（節錄）：
- 人類：王城 `cap`（工事 2.5），城市 `cityA`（工事 1.5，駐軍 30），其控制地塊與魔物部落 `tribeX` 接壤。
- 魔物：主巢 `nest`、副巢 `subA`、部落 `tribeX`（工事 1.0，駐軍 50）。魔物 AI 想吃 `cityA` 拿奴隸。
- `cityA` 與 `tribeX` 之間隔一條河，目前**僅一座窄橋** `ford1` 可通。

**① 內政階段**
- 人類提交：`RESEARCH` 偵察科技「斥候網」（+0.2 clarity）、`RECRUIT` 2 隊精兵到 `cityA`、
  `FORTIFY` 在 `cityA` 面河側築 `wall`（進建造佇列，工期 2 月）。
- 魔物提交：`ALLOCATE` 把奴隸轉兵力、`RECRUIT` 大批哥布林到 `tribeX`、`BUILD_BRIDGE` 在河上游
  另架一座橋 `br2`（工期 1 月，想開第二進攻軸線）。
- 結算：人類 `intel_clarity_bonus` +0.2、`cityA` 駐軍 30→55；`tribeX` 駐軍 50→90；兩件建造入佇列。

**② 情報階段**
- 人類探查 `tribeX` 所在地塊：clarity 升至 0.7 → `is_accurate=True`，加噪後真值 90 → 觀測區間
  `(78,102)`。玩家大致知道 tribeX 變強，但**沒看到上游正在架的 `br2`**（在視野半徑外）。
- 魔物探查 `cityA`：clarity 僅 0.3 → 命中誤導，低報為 `(20,35)`，AI 誤以為 cityA 仍虛弱。

**③ 軍事階段**
- 人類（知 tribeX 變強）保守：`cityA` 守軍 `DEFEND`，從王城 `REINFORCE` 1 隊精兵（軍團沿 `road`
  快速抵達，移動點足夠）。
- 魔物（誤信情報、且 `br2` 本月完工）大膽：主力沿 `ford1` 正面、一支偏師沿新橋 `br2` 包抄 → 兩路 `ATTACK` `cityA`。

**④ 戰鬥結算（即時戰術）**
- `cityA` 接戰，`battle_seed=derive_seed("demo","battle",cityA_coord,3,0)`：
  - 切出 `BattleMap`：含河、`ford1` 窄橋、`br2` 新橋，但 `cityA` 的 `wall` **尚未完工（還在佇列）**，
    本場戰場沒有牆。
  - 雙方展開為 unit group。玩家 `INTERACTIVE`：把精兵列陣守住 `ford1` 橋頭（地形瓶頸 → 守方
    `terrain_defense` 高），分一隊機動單位攔截 `br2` 包抄的哥布林。
  - 即時模擬：哥布林 `horde` 在橋頭展不開隊形、傷亡慘重；包抄偏師被機動隊纏住。
  - 結束：魔物潰逃，`outcome=repelled`，`cityA` 仍屬人類；精兵小損、菁英獲 `xp`。
- 突發事件擲值 → 魔物主巢獲少量補員。寫入 `combat_log` / `BattleReplay`。

**月末 RESOLVE**：`cityA` 的 `wall` 佇列 `turns_remaining` 2→1（下月才完工）；魔物 `br2` 已於本月完工
（留待下次再戰時改變戰場）。王城與主巢都未易主 → `winner=None`，`month` → 4。

**驗證重點**：偵察投資讓人類看清守軍卻沒看到上游建橋（視野半徑限制）、魔物用 `BUILD_BRIDGE`
主動開新進攻軸線（改造環境創造戰術）、玩家在即時戰場利用橋頭瓶頸以寡擊眾（戰術微操）、
未完工的牆證明建造佇列跨月生效、所有亂數源自 `derive_seed` 可重現——各系統如設計般串起來。
