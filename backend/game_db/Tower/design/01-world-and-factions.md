# 01 — 世界觀與陣營

## 世界設定

異世界、傳統劍與魔法。世界上的**人類只有女性**，且天生擁有強大的基因與魔法天賦——
單兵素質遠勝魔物，但人口稀少、難以量產。人類聚居於少數高度設防的城市，以**王城**為核心。

與人類為敵的是**魔物**：種類繁多、繁殖快速、組織鬆散但數量龐大，棲息於地形複雜的
**巢穴**。魔物個體弱，但以量、以變異、以滲透取勝。

這是一場**質 vs 量**、**少數堅城 vs 多數脆窩**的非對稱戰爭。

## 兩大陣營（非對稱設計）

| 面向 | 人類方（玩家） | 魔物方（AI） |
|------|----------------|--------------|
| 核心據點 | 王城（唯一，被佔即敗） | 魔巢 / 主巢（被佔即敗） |
| 次要據點 | 城市 A~C | 副巢、部落（一個巢穴 = 部落+副巢叢集） |
| 據點特性 | 少而堅；高工事、高防禦倍率 | 多而脆；易滲透、冗餘高、地形複雜 |
| 戰力來源 | 精兵：高單體數值、可被科技強化 | 群體：低單體數值、靠數量與變異 |
| 經濟資源 | 戰鬥資源、研究點 | 奴隸、魔物源 |
| 取得方式 | 佔領魔巢 → 戰鬥資源 | 佔領人類城 → 奴隸（可轉兵力或魔物源） |
| 成長路線 | 科技樹（基因 / 武器 / 工事 / 偵察） | 進化樹（以魔物源解鎖更強物種） |
| 致命弱點 | 只有一座王城＝單點失效 | 主巢一旦失守即全盤崩潰，但外圍冗餘多 |

設計意圖：人類玩家必須「以寡擊眾、層層設防」，承受不起王城風險；魔物 AI 則「廣鋪
多路、消耗滲透」，可以用部落 / 副巢當消耗品換取主力突破。

## 魔物物種圖鑑（可擴充）

物種定義放在 [`../data/monsters.yaml`](../data/monsters.yaml)，以 `MonsterSpecies` 模型驗證。
新增物種**只需改 YAML，不必改 schema**。初始圖鑑：

| 物種 | 定位 | 特性 (traits) | 進化去向 |
|------|------|---------------|----------|
| 哥布林 goblin | 廉價雜兵、走量 | `horde`（數量加成） | 半獸人 |
| 河童 kappa | 水域單位 | `amphibious`（水/沼地形加成） | — |
| 史萊姆 slime | 肉盾、難殲滅 | `split`（受擊分裂） | — |
| 半獸人 orc | 中堅主力 | `frontline` | 惡魔 |
| 惡魔 demon | 高階精英 | `caster`（魔法）、`fear` | — |

擴充範例（驗證資料/規則解耦）：在 `monsters.yaml` 新增「巨魔 troll」`{base_hp 高, traits: [regen]}`
即可進入孵化池，無需動程式。詳見 [05-tech-and-evolution](05-tech-and-evolution.md)。

## 勝負條件

- **人類勝**：佔領魔物主巢（`node_type=main_nest`），或殲滅全部巢穴。
- **人類敗**：王城（`node_type=capital`）被魔物佔領。
- 反之為魔物方勝負條件。`GameState.winner` 記錄結果，`None` 表示進行中。

相關模型：`Side`、`NodeType`、`Faction`、`GameState.winner`（見 [`../schema/models.py`](../schema/models.py)）。
