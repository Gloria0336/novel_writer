# 07 — 魔物方 AI 對手

魔物方由**規則型 AI** 操控（`Faction.is_ai=True`）。AI 在四階段各自做決策，
且**只能依不完全資訊行動**——它和玩家一樣受戰爭迷霧限制（[08](08-intelligence-and-fog.md)），
只能用自己在情報階段取得的 `IntelReport` 推測人類部署。

## 盤面評估函數（共用）

AI 各階段決策都呼叫一個簡單評估器，對每個相關據點打分：

```
threat(node)   = 鄰接敵軍估計戰力總和 / (node 守軍 × fortification × terrain_defense)
value(node)    = 加成價值 + 類型權重（主巢 >> 副巢 > 部落；王城 / 城市對攻擊方的價值）
weakness(enemy_node) = enemy 估計守軍 / (己方可投射戰力)   # 越低越好打
```

估計值來自情報（可能有誤），因此 AI 會「賭錯」——這是設計的一部分。

## 各階段 AI 行為

### 內政階段
依資源與局勢在兩條路線間取捨：
- **走量**：奴隸 → 兵力，孵化大量雜兵，適合領先 / 想滾雪球壓制時。
- **走質**：奴隸 → 魔物源，投入進化解鎖高階物種，適合僵持 / 需質變突破時。
- 同時保留部分魔物源做每月孵化補員；威脅高時優先補受威脅巢穴的兵力。

### 情報階段
優先探查：① 威脅自身主巢 / 副巢的人類鄰接據點；② 計畫進攻的人類城（先偵察守軍）。
偵察科技不足時，AI 會分散探查以降低被單一錯誤情報誤導的風險。

### 軍事階段
優先序：
1. **防守**：`threat` 超過閾值的巢穴 → `DEFEND` / 從鄰巢 `REINFORCE`。
2. **進攻**：挑 `weakness` 最低（最好打）且 `value` 高的人類據點 → 多路 `ATTACK`，
   可用部落 / 副巢的雜兵當消耗品包抄。
3. **擴張**：行有餘力佔中立地形（搶加成）。
4. 其餘 `HOLD`。

### 戰鬥結算
被動承受結算；結果回饋下一個月的評估（失守 → 提高該區防守權重）。

## 難度旋鈕（預留）

- 情報清晰度上限、評估閾值、走量/走質傾向、每月行動數，都可參數化成難度設定。
- 因 AI 與玩家共用同一套「提交 → 結算」介面（[06](06-turn-and-phases.md)），
  未來要把魔物方改成第二位人類玩家（雙人對戰）只需替換提交來源，引擎不變。

本階段只定義 AI 的**決策框架與優先序**，不實作評估器程式；閾值與權重留待平衡。
