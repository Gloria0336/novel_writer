# 07 — 魔物方 AI 對手

魔物方由**規則型 AI** 操控（`Faction.is_ai=True`）。AI 在四階段各自做決策，
且**只能依不完全資訊行動**——它和玩家一樣受戰爭迷霧限制（[08](08-intelligence-and-fog.md)），
只能用自己在情報階段取得的 `IntelReport` 推測人類部署。

## 盤面評估函數（共用）

AI 各階段決策都呼叫一個簡單評估器，對每個相關結構 / 地塊打分（連續地圖以「可在 N 個移動點內
抵達」界定鄰接）：

```
threat(struct)   = 可投射敵軍估計戰力總和 / (struct 守軍 × fortification × terrain_defense)
value(struct)    = 加成價值 + 類型權重（主巢 >> 副巢 > 部落；王城 / 城市對攻擊方的價值）
weakness(enemy)  = enemy 估計守軍 / (己方可在期限內投射的戰力)   # 越低越好打
reach(target)    = 軍團沿地塊移動成本（含道路 / 橋 / 密道修正）是否在 movement_points 內
```

估計值來自情報（可能有誤），因此 AI 會「賭錯」——這是設計的一部分。

## 各階段 AI 行為

### 內政階段
依資源與局勢在兩條路線間取捨：
- **走量**：奴隸 → 兵力，孵化大量雜兵，適合領先 / 想滾雪球壓制時。
- **走質**：奴隸 → 魔物源，投入進化解鎖高階物種，適合僵持 / 需質變突破時。
- 同時保留部分魔物源做每月孵化補員；威脅高時優先補受威脅巢穴的兵力。
- **建設決策（混合制新增）**：在 `threat` 高的巢穴 `FORTIFY`（牆 / 壕）；在主攻軸線 `BUILD_ROAD`
  加速調兵、`BUILD_BRIDGE` 開新進攻路線；行有餘力 `TERRAFORM`（如清森林去人類隱蔽）。
  建設與招募 / 進化競爭資源預算（[11](11-construction-and-terraform.md)）。

### 情報階段
優先探查：① 威脅自身主巢 / 副巢的人類鄰接據點；② 計畫進攻的人類城（先偵察守軍）。
偵察科技不足時，AI 會分散探查以降低被單一錯誤情報誤導的風險。

### 軍事階段
以軍團移動（`ArmyMove`，受 `movement_points` 與地塊成本限制）執行，優先序：
1. **防守**：`threat` 超過閾值的巢穴 → `DEFEND` / 從鄰巢 `REINFORCE`。
2. **進攻**：挑 `weakness` 最低（最好打）、`value` 高且 `reach` 可達的人類結構 → 多路 `ATTACK`，
   可走密道包抄、用部落 / 副巢的雜兵當消耗品。
3. **擴張**：行有餘力佔中立 / 資源地塊（搶加成）。
4. 其餘 `HOLD`。

### 戰鬥結算（戰術層 AI）
魔物方戰鬥走 `AUTO_RESOLVE`：由**戰術 AI** 在即時引擎上下 unit-group 指令
（[10](10-tactical-combat.md)），框架為：
- **行為樹 / utility**：依戰場態勢選「集火最脆群」「保護首領（菁英）」「以 horde 群包夾」
  「守城牆 / 守橋」「潰逃保存」等行為，產出陣型 / 移動 / 攻擊 / 技能指令。
- 善用戰場地形與己方先前的建設（牆 / 壕 / 橋 / 改過的地形）。
- 結果回饋下一個月戰略評估（失守 → 提高該區防守 / 建設權重）。

## 難度旋鈕（預留）

- 情報清晰度上限、評估閾值、走量/走質傾向、每月行動數，都可參數化成難度設定。
- 因 AI 與玩家共用同一套「提交 → 結算」介面（[06](06-turn-and-phases.md)），
  未來要把魔物方改成第二位人類玩家（雙人對戰）只需替換提交來源，引擎不變。

本階段只定義 AI 的**決策框架與優先序**，不實作評估器程式；閾值與權重留待平衡。
