# Gossips Database

> 收錄尚未確認真實性、來源可疑、或角色之間流傳的小道消息。

本資料夾不是正史資料庫。任何條目在被其他資料確認前，都只代表「有人這樣說過」或「某處正在流傳這個說法」，不能直接當作世界觀事實使用。

## 收錄範圍

- 城市、地區、派系、人物之間流傳的傳聞。
- 尚未被官方紀錄、角色親眼確認、或可靠文本證實的事件。
- 可能含有誇大、誤傳、政治宣傳、恐慌、迷信、故意放出的假消息。
- 對劇情有用，但目前還不適合放進 `bible/`、`regions/`、`factions/` 或 `updates/daily/` 的資訊。

## 不收錄

- 已確認為正史的設定。
- 純作者筆記、待辦事項、寫作靈感。
- 需要永久保密的真相或幕後設定；這類內容應放入 `context/secrets-lockbox.md`。
- 已經有明確日期與可靠見證的正式事件；這類內容應放入 `updates/daily/`。

## 建議檔名

單則傳聞可獨立成檔，建議使用：

```text
Y000-M01-D01-short-title.md
```

若消息沒有明確日期，可使用：

```text
undated-short-title.md
```

## 條目格式

```markdown
# 傳聞標題

- `Rumor ID`: `rumor-000`
- `First Heard`: `Y000-M01-D01` 或 `unknown`
- `Location`: 傳出地點
- `Source`: 消息來源或流傳人群
- `Reliability`: `unknown` / `low` / `medium` / `high`
- `Status`: `unverified` / `confirmed` / `false` / `distorted` / `obsolete`
- `Related`: 相關人物、派系、地區、事件

## 傳聞內容

用傳聞本身的語氣描述，保留模糊、矛盾、誇張或情緒化的地方。

## 已知線索

- 支持此傳聞的跡象。
- 反駁此傳聞的跡象。
- 尚待確認的疑點。

## 可能影響

- 若為真，可能牽動哪些角色、派系、地區或事件。
- 即使為假，是否會造成恐慌、衝突、誤判或政治效果。

## 後續處理

- 需要在哪些日誌、章節、人物行動或地區紀錄中驗證。
- 若確認為真，應轉寫到哪個正史檔案。
```

## 可信度標記

- `unknown`: 只有傳聞本身，尚無可判斷依據。
- `low`: 來源偏頗、內容誇張、或與既有資料衝突。
- `medium`: 有部分跡象支持，但仍缺少關鍵證據。
- `high`: 多方來源吻合，但尚未正式確認為正史。

## 狀態標記

- `unverified`: 尚未確認。
- `confirmed`: 已確認為真，應補進對應正史資料。
- `false`: 已確認為假。
- `distorted`: 核心可能為真，但細節被誤傳或扭曲。
- `obsolete`: 曾經有用，但已被後續事件取代。

## 使用原則

- 寫作時可以讓角色依據傳聞行動，但不要讓敘事旁白直接把傳聞當真。
- 若傳聞後來被證實，請在原條目保留紀錄，並新增連結到被更新的正史檔案。
- 若傳聞後來被推翻，也請保留條目，因為「錯誤消息如何流傳」本身可能是劇情線索。
- 同一事件若有多個版本，可以分成多個條目，或在同一條目下用來源區分版本。
