# 星落戰紀 - 卡牌遊戲設計文件 v3.4

> 基於 `novel_02` 世界觀的單人 PvE 種族戰爭卡牌遊戲。  
> 本版對齊目前遊戲實作內容，資料來源以 `app/src` 的 TypeScript 定義為準。  
> 類型：種族框架 + 職業框架 + 個體英雄 + 兵力/器具戰場。  
> 目前已實作模式：單場戰鬥、30 層試煉塔。  
> 目前玩家可用卡：170 張，另有敵方/內部卡與惡魔敵方卡。

---

## A. 版本定位

### A.1 v3.4 目標

v3.4 將 v3.3 的設計草案整理成「目前遊戲實作規格」。舊文件中仍保留的短式展示代號，在目前程式中已改為正式卡牌 ID，例如 `T_c_01`、`T_s_35`、`S_l_03`。本文件全面使用實作 ID。

本版同步納入以下已實作內容：

- `device` 獨立卡型：魔導器具仍佔兵力欄，但具有型態、升級、反應、重建等專屬機制。
- 玩家可用卡池 170 張：通用 104 + 六族玩家種族卡 60 + 中立傳說 6。
- 7 位可選英雄：露露、山獵人、芮卡、艾爾諾、曇、艾德·圖林、艾拉·芙萊爾。
- 7 種族框架：人類、精靈、矮人、妖族、獸族、半神族、惡魔。
- 7 職業框架：指揮官、法師、鍛造師、幻術師、狂戰士、神官、冒險家。
- 6 巢穴 + 6 Boss + 30 層試煉塔。
- 次元壁穩定度、腐化階段、次元滲透裂縫與 `T_s_35` 到 `T_s_40` 滲透體。

### A.2 設計支柱

1. **角色即英雄**：英雄卡直接對應小說角色；種族和職業給框架，英雄個體給真正玩法。
2. **戰場是核心**：兵力與器具佔用欄位，場面交換、守護、攻城、裂縫位共同決定壓力。
3. **雙軌攻擊**：兵力遵循兵力優先；行動卡、法術、技能遵循守護優先或效果指定。
4. **資源多軌**：魔力控制回合節奏，鬥志控制英雄技能，種族量表決定族群特色。
5. **腐化是風險槓桿**：穩定度低落不只懲罰玩家，也會打開可被佔據的裂縫資源。
6. **PvE 可擴展**：巢穴、Boss、試煉塔、AI Profile 都是可追加內容的資料層。

### A.3 目前未實作或僅規劃

以下內容不作為 v3.4 已實作規格：

- 內戰模式、種族挑戰模式、PvP。
- Boss 多階段 HP 門檻切換能力。Boss 資料型別有 `phases` 欄位，但目前註解與資料顯示 v1 不啟用多階段。
- 天象事件在塔資料層已定義並抽取，但戰鬥規則層尚未完整套用所有天象效果。
- 完整卡表文件 `card_game_cardlist_v1.1.md` 已修正 ID，但其內容仍偏 v1.1 文件；目前程式卡池以 `app/src/data/cards` 為準。
- `card_upgrade.md` 的場地空間管理升級包已整合到本文件 §M，定位為 Stage 1 規劃內容；它不改變本版「目前已實作卡池 170 張」的統計。

## B. 遊戲流程

### B.1 App 流程

目前入口流程：

1. **英雄選擇**：玩家從 `HERO_LIST` 選 1 位英雄。
2. **模式選擇**：進入單場戰鬥或試煉塔。
3. **單場戰鬥**：選擇 1 個敵人，啟動一場 Battle。
4. **試煉塔**：使用英雄初始牌組，連續挑戰 30 層，勝利後選獎勵並進下一層。

### B.2 戰鬥建立

戰鬥由 `createBattle` 建立，包含：

- 玩家英雄實例與初始牌組。
- 敵方英雄或巢穴實例。
- 雙方牌庫、手牌、墓地、兵力欄。
- 場地、穩定度、腐化階段、裂縫狀態。
- 隨機種子與戰鬥 log。

預設玩家先攻。`BattleState.turn` 只在輪回玩家時增加，因此一個玩家回合與一個敵方回合共用同一回合數。

### B.3 回合結構

```
回合開始
  抽 1 張
  魔力依回合數回復到上限
  結算場地、裝備、滿量表、量表回合開始、幻術師幻影
  重置兵力攻擊狀態、凍結倒數、鍛造師每回合限制

主要階段
  部署兵力或器具
  部署兵力到裂縫位
  施放法術
  使用行動卡
  裝備
  放置場地
  指揮兵力攻擊
  使用英雄主動技能或終極技
  鍛造師改造/製造

回合結束
  結算兵力/器具回合結束效果
  遞減 buff、關鍵字 buff、狀態、英雄能力凍結
  tick 次元裂縫震動倒數
  切換行動方
```

### B.4 勝負條件

| 條件 | 結果 |
|------|------|
| 敵方英雄或巢穴 HP 歸零 | 玩家勝利 |
| 玩家英雄 HP 歸零 | 玩家失敗 |
| 次元壁穩定度歸零 | 玩家失敗 |

## C. 戰場規則

### C.1 場地佈局

每方預設 5 個兵力欄。兵力與魔導器具都佔用同一組 `troopSlots`。場地卡全場同時只能存在 1 張，後放置者會覆蓋舊場地。

穩定度跌破 75 時，雙方各移除最後 1 個兵力欄。若欄位內已有單位，實作直接移除欄位。

### C.2 兵力攻擊

兵力攻擊採「兵力優先」：

- 敵方仍有兵力時，不能直接攻擊英雄或巢穴。
- `siege` 攻城可無視兵力優先，直接打英雄或巢穴。
- 部署當回合有暈眩，不能攻擊。
- `rush` 突進可在部署當回合攻擊兵力。
- `haste` 疾走可在部署當回合攻擊任意合法目標。
- `frozenTurns > 0` 的兵力不能攻擊。
- 兵力互打時雙方交換傷害；防守方視為反擊，每回合每張兵力最多反擊 1 次。同一張兵力在同一回合受到多次攻擊時，第二次起只承受傷害，不再反擊。
- 攻擊英雄或巢穴時只有攻擊方造成傷害。

### C.3 英雄行動與技能

行動卡、技能、終極技使用「守護優先」：

- 敵方有 `guard` 守護兵力時，普通單體敵方目標必須選守護兵力。
- `ignoreGuard` 效果可繞過守護。
- `untargetable` 狀態不可被選取。
- `taunt` 狀態優先度高於守護。
- `marked` 和 `taunt` 可讓目標成為合法優先目標。

### C.4 傷害與防禦

英雄和兵力都有 DEF。一般傷害先扣 DEF，再造成 HP 損失。實作保留最低傷害邏輯，並支援：

- `ignoreDef`：無視 DEF。
- `pierce`：兵力攻擊時無視 DEF。
- 護甲：英雄護甲先吸收傷害。
- `lifesteal`：攻擊造傷後依規則恢復。
- 必殺：`lethal` 兵力對兵力造成有效傷害後直接摧毀目標。
- 滿量表與被動可提供傷害增幅或減免。

### C.5 效果引擎

效果引擎由 `core/types/effect.ts` 定義資料模型，`core/effects/registry.ts` 負責執行，並由 `turn/reducer.ts`、`turn/phases.ts`、`combat/attack.ts`、`combat/damage.ts` 串接到實際戰鬥流程。所有卡牌、英雄技能、終極技、場地、器具反應與巢穴/Boss 光環，最後都會轉成 `Effect[]` 或 `scripted` 標籤進入同一條執行管線。

#### C.5.1 執行入口與來源

`executeEffects(effects, ec)` 逐一執行效果。`EffectContext` 會帶入目前 `BattleState`、資料查詢用 `BattleContext`、施放方 `sourceSide`、來源類型 `sourceKind`、來源實體 `sourceInstanceId` 與來源卡牌 `sourceCardId`。

`sourceKind` 目前包含：

| 來源 | 說明 |
|------|------|
| `troop_play` | 兵力或器具入場效果 |
| `troop_destroy` | 兵力或器具謝幕效果 |
| `spell` | 法術牌效果 |
| `action` | 行動牌效果 |
| `equipment` | 裝備入場效果 |
| `field` | 場地效果 |
| `skill` | 英雄主動技能 |
| `ultimate` | 英雄終極技 |
| `passive` | 回合鉤子、光環、反應、被動腳本 |

每個效果結算後會同步滿量表 buff 與動態被動光環；若戰鬥已結束，後續效果停止執行。這讓連鎖傷害、召喚、死亡清掃、光環重算可以在同一套狀態上連續生效。

#### C.5.2 卡牌鉤子

不同卡型接入效果引擎的位置不同：

| 卡型 | 鉤子 |
|------|------|
| `troop` | `onPlay`、`onDestroy`、`onTurnStart`、`onTurnEnd`、`passive` |
| `device` | 同 troop，另有 `onReaction`、`form`、`upgradeable` |
| `action` | `effects`，可有 `postEffects` |
| `spell` | `effects` |
| `equipment` | `onPlay`、`passive`，裝備數值直接套到英雄 |
| `field` | `effects`，並由回合開始流程處理持續場地 |
| 英雄技能 | `actives[].effects`、`ultimate.effects` |
| 敵方 | `onBattleStart`、巢穴 `auraTags`、AI 行動 |

部署兵力或器具時，會先扣費與放入欄位，再觸發量表、裝備被動、入場效果與死亡清掃。召喚效果 `summon` 只建立單位，不觸發該單位 `onPlay`，用來區分「部署」和「召喚」。

#### C.5.3 效果 DSL

目前基礎效果種類如下：

| `kind` | 功能 |
|--------|------|
| `damage` | 對目標造成傷害，支援 `ignoreDef`、`ignoreGuard`、`lifesteal` |
| `heal` | 治療英雄或兵力，受治療倍率與場地修正 |
| `draw` | 從牌庫抽指定張數 |
| `discard` | 從手牌隨機/頂端棄指定張數，目前實作為移除手牌前段 |
| `summon` | 召喚指定卡牌到指定方空兵力欄 |
| `gauge` | 增減指定方種族量表 |
| `morale` | 增減來源方鬥志 |
| `mana` | 增減魔力，可標為臨時魔力 |
| `armor` | 增加英雄護甲 |
| `buff` | 對英雄或兵力套用 HP/ATK/DEF/CMD 修正 |
| `addKeyword` | 對兵力新增臨時或永久關鍵字 |
| `addStatus` | 對英雄或兵力新增狀態 |
| `freeze` | 對兵力設定 `frozenTurns`，可顯示為冰凍、定身、束縛、暈眩、控制等 |
| `freezeHeroAbility` | 凍結英雄能力面向：行動牌、法術牌、兵力牌/巢穴生成、魔力回復 |
| `stability` | 增減次元壁穩定度，並套用腐化階段與裂縫開啟 |
| `search` | 從牌庫搜尋符合條件的卡加入手牌，目前以卡型為主 |
| `destroyField` | 摧毀目前場地 |
| `scripted` | 交給註冊表中的自訂處理器 |

#### C.5.4 目標解析

目標由 `TargetSelector` 表示：

| selector | 說明 |
|----------|------|
| `self` | 來源方英雄 |
| `playerHero` | 玩家英雄 |
| `enemyHero` | 敵方英雄 |
| `single` | 單一目標，需要 `pickedInstanceId` |
| `all` | 所有符合 filter 的目標 |
| `random` | 從符合 filter 的目標取指定數量，目前解析層回傳候選池前段 |

`TargetFilter` 可限制陣營、實體類型、關鍵字、HP 百分比。陣營可用 `player`、`enemy`、`self`、`all`；實體可用 `hero`、`troop`、`any`。單體敵方目標會再經過守護/嘲諷/不可選中檢查。

單體效果的合法性由 `validateSingleTargetEffects` 與 `canActionTarget` 檢查：

- 敵方有 `taunt` 狀態時，必須先指定嘲諷目標。
- `marked` 與 `taunt` 都可成為優先合法目標。
- 敵方有 `guard` 兵力時，普通單體行動/法術/技能必須指定守護兵力。
- `ignoreGuard` 或本回合 `ignoreGuardThisTurn` 可繞過守護。
- `untargetable` 不可被單體指定，但不擋全體效果。
- 帶有 `UNTARGETABLE_BY_SPELL` 被動標籤的兵力不可被敵方法術單體指定。

#### C.5.5 數值公式

`AmountExpr` 將傷害/治療數字資料化：

| 公式 | 說明 |
|------|------|
| `const` | 固定值 |
| `atk` | 來源英雄 ATK，可乘倍率與加 bonus |
| `spellsCastThisGame` | 本場施法次數乘倍率 |
| `spellsCastThisTurn` | 本回合施法次數乘倍率 |
| `alliesOnBoard` | 己方存活兵力數乘倍率，可加 bonus |
| `rage` | 來源英雄量表值乘倍率，可加 bonus |
| `command` | 來源英雄量表值乘倍率，可加 bonus |

傷害和治療在基礎值算出後，會依來源與場面套用倍率。法術受精靈共鳴、滿量表法術倍率、下張法術翻倍、部分場地影響；行動卡受滿量表行動倍率、狂戰士狂暴、指揮官號令、裝備與場地影響。治療受神官祝福、滿量表法術倍率、場地與腐化治療倍率影響。

#### C.5.6 傷害、治療與死亡清掃

傷害流程分成基礎傷害與被動修正：

1. 解析目標與基礎數值。
2. 套用來源倍率。
3. 單體敵方目標檢查守護、嘲諷、不可選中。
4. 對英雄或兵力套用被動防禦。
5. 一般傷害先扣 DEF；`ignoreDef`、`pierce` 或固定傷害可跳過 DEF。
6. 英雄護甲先吸收非固定傷害。
7. 滿量表、神域結界、神隱、當回合減傷等再修正英雄實際受傷。
8. 傷害結束後處理汲取、鬥志、量表、裝備被動與死亡清掃。

死亡清掃由 `reapDeadTroops` 統一處理。它會反覆掃描兵力欄與裂縫佔據者，直到沒有新的死亡單位，最多遞迴 8 層避免無限連鎖。死亡時會：

- 觸發該兵力或器具的 `onDestroy`。
- 把卡片放入墓地；器具額外放入 `destroyedDevices`，供重建效果使用。
- 擊殺敵兵給擊殺方鬥志；擊殺裂縫滲透體給較低的裂縫擊殺鬥志。
- 己方兵力死亡給己方鬥志補償。
- 觸發種族量表的自家兵力死亡增量。
- 讓裂縫佔據者死亡後騰出裂縫。
- 觸發器具 `allyTroopDestroyed` 反應。
- 重算動態被動光環。

治療會依目標封頂到 `maxHp`。場地可禁止或降低兵力治療，腐化場地可降低非擁有方治療。

#### C.5.7 Buff、狀態與持續時間

持續時間分三種：

| duration | 實作 |
|----------|------|
| `permanent` | 以 9999 回合表示 |
| `thisTurn` | 設為 1，於該方回合結束倒數並移除 |
| `turns` | 指定回合數 |

`buff` 會立即改變英雄或兵力的數值，並寫入 `buffs`；到期時回復數值。HP buff 同時改 `hp` 與 `maxHp`，移除時會把目前 HP 限制回新的 `maxHp`。

`addKeyword` 只套用在兵力，寫入 `keywordBuffs`。臨時關鍵字到期時，若該關鍵字不是卡牌原生關鍵字、也沒有其他仍有效的關鍵字 buff，才會移除。

`addStatus` 支援：

| 狀態 | 實作效果 |
|------|----------|
| `taunt` | 嘲諷，敵方單體攻擊/效果必須優先指定 |
| `marked` | 針對，可成為優先合法目標 |
| `untargetable` | 不可被單體指定 |
| `invincible` | 允許 DEF 把傷害壓到 0 |

負面 buff、負面狀態與 `freeze` 會受到部分淨化/免疫效果影響，例如 `S_c_14` 的 purify 選項會移除負面 buff 與凍結，並給本回合後續負面狀態免疫。

#### C.5.8 關鍵字

| ID | 中文 | 實作效果 |
|----|------|----------|
| `guard` | 守護 | 英雄行動卡、法術、技能等單體敵方目標必須優先指定守護兵力 |
| `rush` | 突進 | 部署當回合可攻擊敵方兵力 |
| `haste` | 疾走 | 部署當回合可攻擊任意合法目標，包括英雄/巢穴 |
| `lethal` | 必殺 | 兵力交戰造成有效傷害後直接摧毀敵方兵力 |
| `lifesteal` | 汲取 | 兵力關鍵字已保留；目前主要汲取結算走 `damage.lifesteal` 或裝備被動 |
| `menace` | 威壓 | 不可被敵方兵力指定為攻擊目標，除非目標同時具備優先狀態 |
| `pierce` | 穿透 | 兵力攻擊時無視 DEF |
| `siege` | 攻城 | 無視兵力優先制，可直接攻擊英雄/巢穴 |
| `frozen` | 凍結 | 標籤層定義；實際控制由 `frozenTurns` 與 `frozenDisplayName` 執行 |

職業關鍵字不在 `Keyword` 集合內，而是由職業資料和效果管線讀取：

| 職業關鍵字 | 目前接入 |
|------------|----------|
| `command` | 指揮官行動卡傷害依己方場上兵力數提升 |
| `overload` | 法師/超載職業本回合第 3 次起施法給臨時魔力 |
| `forge` | 鍛造師可用 `FORGE_ACTION` 製造裝備或器具 |
| `phantom` | 幻術師回合開始有空欄時召喚幻影 |
| `berserk` | 狂戰士行動卡傷害依已損 HP 提升 |
| `blessing` | 神官治療量提升 |

#### C.5.9 回合旗標

部分短期效果不寫入卡牌或英雄 buff，而寫入 `TurnFlagsState`，在回合開始重置：

| 旗標 | 用途 |
|------|------|
| `extraActionsThisTurn` | 本回合可額外使用行動卡 |
| `actionDisabledThisTurn` | 本回合禁止再使用行動卡 |
| `firstAttackDoubleInstanceIds` | 指定兵力首次攻擊傷害翻倍，觸發後消耗 |
| `actionCardsPlayedThisTurn` | 行動卡使用次數計數 |
| `ignoreGuardThisTurn` | 本回合單體行動/技能可無視守護 |
| `deployDiscount` | 本回合部署折扣 |
| `nextSpellDouble` / `currentSpellDouble` | 下一張法術翻倍與當前法術翻倍狀態 |
| `nextEquipDiscount` | 下一張裝備折扣 |
| `packTacticsActive` | 行動命中時額外累積血怒 |

#### C.5.10 反應系統

器具的 `onReaction` 由 `core/effects/reactions.ts` 驅動，不使用全域事件匯流排，而是在 reducer 與死亡清掃的固定 hook 點掃描場上的 device。

目前事件：

| 事件 | 反應方 |
|------|--------|
| `enemySpellCast` | 施法者的對立方 |
| `enemyHeroAttacked` | 被攻擊英雄的一方 |
| `allyTroopDestroyed` | 陣亡兵力同陣營 |
| `enemyEquipmentPlayed` | 裝備使用者的對立方 |

反應深度最多 8 層，並用 `(instanceId:event)` lock 避免同一來源在同一事件中反覆觸發。凍結中的器具不會反應。

#### C.5.11 Scripted 註冊器

`scripted` 是效果引擎的擴充層。`ensureScriptedRegistered()` 會依序註冊：

1. `registerCoreScripted()`：通用卡、裂縫、部分巢穴/Boss、通用裝備標籤。
2. `registerHeroScripted()`：玩家英雄終極技、曇、艾拉、幻術師職業生成。
3. `registerRaceCardScripted()`：六族玩家卡、中立傳說、惡魔卡。
4. `registerDeviceScripted()`：魔導器具專屬效果。

同名 tag 後註冊者會覆蓋先前處理器。實作上英雄版 `TOTAL_MOBILIZATION`、`PRIMAL_AWAKENING` 會覆蓋 core 中較舊的簡化版本。

#### C.5.12 動態被動與目前簡化點

動態被動由 `syncDynamicPassiveAuras` 每次效果後與動作後重算，會先移除上一輪 `DYNAMIC_PASSIVE` 來源 buff，再依目前場面重新套用。已實作的動態項包含：

- `ATK_PER_ALLY`：依其他己方兵力數加 ATK。
- `MOUNTAIN_KING_GUARD`：依己方裝備數加 DEF。
- `Y_SERPENT_GUARD`：妖形時加 ATK，非妖形時降低所受法術傷害。
- `DM_CURSE_GENERAL_AURA`：敵方詛咒將領在場時，對方兵力 ATK/DEF -2。
- `HERO_DEF_PLUS_2_WHILE_ALIVE`：特定器具存活時英雄 DEF +2。
- `CLAN_WARHAMMER`：依己方器具數提高英雄 ATK。

目前仍有部分效果採 MVP 簡化：

- `random` 目標解析尚未在 `resolveTargets` 內真正抽樣。
- 部分場地 tag 本身是 no-op，實際效果分散在回合開始、傷害倍率或場地專屬處理。
- 部分 passive tag 只是標記，由 reducer、damage、phases、dynamic passives 在對應節點讀取。
- 兵力關鍵字 `lifesteal` 的戰鬥回血尚未完整實作為兵力自身回血，主要汲取效果由 `damage.lifesteal` 與裝備被動處理。
- 部分 scripted 註解仍標示 MVP 略過，但對應核心玩法多已在其他管線補上，例如法術不可指定、法術傷害免疫、裝備被動、動態光環。

## D. 資源系統

### D.1 魔力

魔力上限通常隨回合成長，最高 10。精靈種族框架的絕對上限是 13。回合開始時魔力回滿，臨時魔力由效果額外給予。

部分效果會改變費用：

- 矮人滿爐火：裝備與器具費用 -1，最低 1。
- 精靈場地 `F_e_01`：兵力與器具部署費用 +1。
- 矮人場地 `F_dw_01`：己方裝備費用 -1。
- 人類動員、裝備折扣、鍛造效果可造成局部費用修正。

### D.2 鬥志

鬥志上限 100，用於英雄技能與終極技。

| 來源 | 數值 |
|------|------|
| 擊殺一般敵方兵力 | +15 |
| 行動卡命中 | +10 |
| 我方兵力被摧毀 | +5 |
| 擊殺裂縫滲透體 | +10，與一般擊殺互斥 |

終極技消耗 100 鬥志，每場戰鬥每位英雄只能施放 1 次。

### D.3 種族量表

量表由種族框架定義名稱、上限與滿量表效果，再由英雄個體定義觸發來源。已支援來源：

- 兵力進場。
- 兵力每回合存活。
- 我方兵力被摧毀。
- 法術施放。
- 裝備打出。
- 器具打出。
- 回合開始。
- 英雄受傷跨過百分比門檻。
- 鍛造師改造/製造。

當量表等於有效上限時，該種族的滿量表 buff 自動同步；花費量表後會移除或重算 buff。

## H. 卡牌系統

### H.1 卡牌 ID 命名規則

所有可索引卡牌 ID 必須符合：

```
/^(A|F|T|S|E)_(c|h|e|dw|f|b|de|g|m|l|s|o)_\d{2}$/
```

第一段代表類型：

- `T`：兵力或器具。器具目前使用 `T_m_*`，但卡牌 `type` 是 `device`。
- `A`：行動。
- `S`：法術。
- `E`：裝備。
- `F`：場地。

第二段代表來源：

| 來源 | 說明 |
|------|------|
| `c` | 通用 |
| `h` | 人類 |
| `e` | 精靈 |
| `dw` | 矮人 |
| `f` | 妖族 |
| `b` | 獸族 |
| `g` | 半神族 |
| `m` | 魔導器具 |
| `l` | 中立傳說 |
| `de` | 惡魔敵方種族卡 |
| `s` | 系統/敵方內部卡、裂縫滲透體 |
| `o` | 保留來源；§M 規劃暫借為職業限定卡容器 |

### H.2 卡牌類型

| type | 中文 | 功能 |
|------|------|------|
| `troop` | 兵力 | 部署到兵力欄或裂縫位的戰鬥單位 |
| `device` | 魔導器具 | 佔用兵力欄，但可有型態、升級、反應、回合自動效果 |

## I. 次元壁與裂縫

### I.1 穩定度與腐化階段

穩定度初始 100，範圍 0 到 100。每次穩定度改變後重新計算腐化階段。

| 穩定度 | 階段 | 實作效果 |
|--------|------|----------|
| 76-100 | 0 | 無腐化效果 |
| 51-75 | 1 | 首次抵達時雙方兵力欄 -1 |
| 26-50 | 2 | 記錄敵方額外部署提示；首次跌破 50 會開裂縫 |
| 1-25 | 3 | 治療效果 ×0.5 |
| 0 | 4 | 玩家直接失敗 |

注意：`openRiftIfNeeded` 的條件是 `stability < 50`，所以穩定度正好 50 時不開裂縫。

### I.2 裂縫生命週期

裂縫為 `BattleState.rift`，未開啟時是 `undefined`。首次穩定度跌破 50 後建立，之後不可逆。

裂縫狀態：

- `open`：空置，回合結束 tick 震動倒數。
- `player`：玩家佔據，震動暫停。
- `enemy`：敵方滲透體佔據，玩家需擊殺或處理。

佔據者獲得：

- ATK ×2。
- DEF +5。
- HP 不變。

佔據者死亡後，裂縫回到 Open，倒數重設為 1。

### I.3 滲透體池

裂縫 Open 時，回合結束倒數歸 0 會召喚 1 個敵方滲透體到裂縫位。每張同場最多召喚 2 次。

| 5-8 | `T_s_35` 到 `T_s_38` | 往上一階，抽 `T_s_35` 到 `T_s_40` |
| 9+ | `T_s_35` 到 `T_s_40` | 不再上升 |

## M. 卡牌升級包：場地空間管理機制

本章整合 `card_upgrade.md`。其定位是 **Stage 1 設計交付物**，主題是「兵力欄滿格時的差異化場地管理」。本章不代表目前程式已實作，也不計入 §H.3 的玩家可用 170 張卡池；若進入實作，需同步更新 `card_game_cardlist`、規則層 reducer/action、AI 與測試。

### M.1 設計目標

### M.2 滿格部署總規則

當己方兵力欄已滿時：

- 預設不能部署新的兵力卡或器具卡；手牌保留。
- 行動、法術、裝備、場地不受兵力欄滿格限制。
- 裂縫位是獨立欄位，不受 5 格兵力欄限制。
- 人類前線第六格是獨立特殊欄位，不等同於一般兵力欄。
- 特定種族或職業可透過本章規劃機制例外騰位、取代、回手或使用特殊欄位。

規劃中的滿格解法：

| 種族/職業 | 解法 | 核心代價 | 是否消耗場上單位 |
|-----------|------|----------|------------------|
| 妖族 | 消耗幻影騰位或原位升級幻影 | 幻影資源、行動費用 | 視卡牌而定 |
| 人類 | 推 1 個兵力到前線第六格 | 前線單位 DEF -2 且承受嘲諷火力 | 否 |
| 獸族 | 血祭部署 | 犧牲己方兵力 | 是 |
| 鍛造師 | 構裝體拆解或升級成魔導器具 | 先鋪構裝體，路線受限 | 是，或取代 |
| 指揮官 | 撤退回手、全線換防、預備陣形 | 魔力/行動費用與時機 | 多數否，A_o_06 是 |
| 惡魔 Boss | 暗黑獻祭/永恆契約 | 自殘風險 | 是或共享傷害 |

### M.3 嘲諷狀態補充

效果引擎已存在 `taunt` 狀態，並已在目標合法性檢查中支援。升級包要求將它正式作為卡牌設計可用狀態：

- 嘲諷強制敵方單體兵力攻擊、行動卡、法術、技能優先指定嘲諷目標。
- 多個嘲諷目標同時存在時，攻擊方可在嘲諷目標中選擇。
- 嘲諷優先級高於守護。
- `marked` 目前也會成為優先合法目標；正式實作升級包時需確認嘲諷與針對同場時的優先級細節。
- 嘲諷與威壓的設計意圖互斥：威壓是不被兵力指定，嘲諷是強制被指定。若同一單位將同時獲得兩者，應由規則層拒絕、移除其一，或明確定義覆蓋順序。

| 狀態 | 獲得嘲諷，直到陣亡或被收回 |
| 消失 | 前線單位陣亡或收回時，第六格關閉 |
| 裂縫關係 | 第六格與裂縫位彼此獨立，可同時存在 |

規劃新卡：

| ID | 卡名 | 費用 | 稀有度 | 效果 |
|----|------|------|--------|------|
| `A_h_05` | 前線推進 | 2 | uncommon | 指定己方 1 名兵力成為前線單位；ATK +3、DEF -2、嘲諷直到陣亡或收回；每場最多 2 次 |
| `A_h_06` | 前線換防 | 3 | rare | 將前線單位收回手牌，保留 buff 與 modifier，並可立即指定另 1 名兵力成為新前線單位 |

滿格時，`A_h_05` 可把一般欄位中的 1 個兵力推到第六格，使原欄位騰出來部署新兵力。

### M.6 獸族：血祭部署

血祭部署是獸族的滿格解法：犧牲弱者餵養強者。它應作為獸族英雄內建規則，而不是單張卡牌獨有能力。

規劃規則：

- 部署 4 費以上兵力時，可消滅 1 名己方兵力作為祭品。
- 若兵力欄已滿，祭品先離場，騰出的欄位可部署新兵力。
- 祭品觸發謝幕曲。
- 主動獻祭不給己方兵力死亡鬥志補償，避免農鬥志。
- 新兵力正常觸發入場曲。
- 4 費以下兵力不可血祭部署。

血祭加成：

| 新兵力費用 | 加成 |
|------------|------|
| 4-5 | ATK +5、HP +5 |
| 6+ | 上述加成，並獲得突進 |
| 7+ | 上述加成，並獲得汲取 |

規劃新卡：

| ID | 卡名 | 類型 | 費用 | 稀有度 | 效果 |
|----|------|------|------|--------|------|
| `A_b_03` | 血祭儀式 | 行動 | 2 | uncommon | 本回合下次血祭部署時，祭品目前 ATK 完整轉移到新兵力，永久加值；每場最多 2 次 |

### M.7 鍛造師：構裝體流派

構裝體是鍛造師的佔位緩衝。它和目前 `device` 魔導器具不同：構裝體是職業行動製造出的構造物，用來拆解、回收、升級成正式魔導器具。

規劃中的構裝體規則：

- 鍛造師可在主要階段主動拆解己方構裝體，回復 2 點本回合可用魔力並抽 1 張。
- 構裝體標記為鍛造構造物。
- 構裝體不觸發謝幕曲。
- 構裝體不可被治療。
- 一般兵力不能部署到構裝體位置來取代它。
- 只有魔導系列 `T_m_*` 可透過構裝升級取代構裝體。

規劃新卡：

| ID | 卡名 | 類型 | 費用 | 稀有度 | 效果 |
|----|------|------|------|--------|------|
| `A_o_01` | 鋼鐵構裝 | 行動 | 1 | common | 部署 1 個鋼鐵構裝體到空兵力欄；10/0/4/守護；不可治療、不觸發謝幕；鍛造師限定 |
| `A_o_02` | 緊急拆解 | 行動 | 0 | uncommon | 消滅己方 1 名構裝體，回復 2 點本回合可用魔力，抽 1；鍛造師限定 |
| `A_o_03` | 構裝升級 | 行動 | 3 | rare | 指定己方構裝體，將其取代為手牌中 1 張 `T_m_*`，該魔導器具永久 HP +5 / ATK +3 / DEF +3 並觸發入場；鍛造師限定 |

與目前實作的關係：目前已存在 7 張 `T_m_*` device；升級包不需要 D 系列遷移，因為正式實作已採 `T_m_01` 到 `T_m_07`。

### M.8 指揮官：撤退換防

指揮官的場地管理哲學是保留卡牌價值。撤退回手不是消滅，因此可以重置受傷單位、重排部署節奏，並避免不必要的謝幕連鎖。

規劃新卡：

| ID | 卡名 | 類型 | 費用 | 稀有度 | 效果 |
|----|------|------|------|--------|------|
| `A_o_04` | 戰術撤退 | 行動 | 3 | uncommon | 將己方 1 名兵力收回手牌；HP 重置為滿，buff 與 modifier 清除；本回合可立即重新部署兵力且費用 -1，最低 0；指揮官限定 |
| `A_o_05` | 全線換防 | 行動 | 5 | rare | 將己方所有兵力收回手牌，HP 重置為滿、buff 清除；抽 2；下回合兵力部署費用 -2，每張最多一次；指揮官限定 |
| `A_o_06` | 預備陣形 | 行動 | 2 | uncommon | 本回合若兵力欄滿，下次部署兵力時自動消滅最左側己方兵力騰位；觸發謝幕曲，獲得鬥志 +5；指揮官限定 |

職業卡跨種族通用。任何具備指揮官職業的英雄都可使用指揮官職業卡；任何具備鍛造師職業的英雄都可使用鍛造師職業卡。`o` 來源代碼在此規劃中暫借為職業卡容器。

### M.9 惡魔 Boss：黑暗契約

本小節來自 `card_upgrade.md` 的魔族 boss 限定法術提案，但提案 ID `S_de_01`、`S_de_02` 已與目前實作中的惡魔卡衝突：

- 目前 `S_de_01` 是「蟲群滲透」。
- 目前 `S_de_02` 是「黑暗詛咒」。

因此下列卡牌若進入實作，必須重新編號，或明確取代既有惡魔卡。文件暫以提案 ID 標示來源。

| 提案 ID | 卡名 | 類型 | 費用 | 稀有度 | 效果 |
|---------|------|------|------|--------|------|
| `S_de_01` | 暗黑獻祭 | 法術 | 3 | uncommon | 消滅施法者己方 1 名兵力 A，指定己方另 1 名兵力 B；B 獲得 A 目前 ATK + 目前 DEF 的永久加值，並失去等同 A 目前 HP 的 HP |
| `S_de_02` | 永恆契約 | 法術 | 4 | rare | 指定己方兵力 A 與對方兵力 B 建立連結，直到任一方死亡；A/B 受到傷害時雙方各承受 X/2；B 每回合 -1 HP 且 ATK -3；同場最多 1 次 |

AI 使用建議：

- 暗黑獻祭：Boss 場上至少 2 名兵力時才施放；優先犧牲 HP 或 ATK 最低者。
- 永恆契約：Boss 與玩家各至少 1 名兵力時才施放；優先連結玩家 ATK 最高的兵力。

### M.10 跨機制交互

| 交互 | 規則意圖 |
|------|----------|
| 幻影 + 血祭部署 | 可用幻影當祭品，但幻影 ATK 低，與 `A_b_03` 的收益自然偏低 |
| 前線單位 + 血祭部署 | 前線單位可作祭品；死亡後第六格消失，新兵力進一般欄位 |
| 前線單位 + 永恆契約 | Boss 連結前線單位可能被玩家反利用，因前線嘲諷會吸火並讓連結方分擔傷害 |
| 構裝體 + 撤退換防 | 構裝體不應被 `A_o_04` 回手；已升級成 `T_m_*` 的魔導器具可依實作決策是否允許回手 |
| `A_o_06` + 謝幕曲 | 可主動引爆最左側己方謝幕曲，形成跨種族 combo |
| 滿格部署 + 裂縫 | 裂縫位獨立，不需要滿格部署機制；構裝體不可進裂縫位 |
